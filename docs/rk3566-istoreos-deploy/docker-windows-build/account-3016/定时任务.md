# Windows 定时任务配置 (account-3016)

本文件说明如何配置 account-3016 的定时阅读任务。

## 方案：Windows 任务计划程序

### 配置步骤

1. 以管理员身份运行 PowerShell

2. 执行定时任务配置脚本：
   ```powershell
   # 创建定时任务（每天 0:00, 6:00, 12:00, 18:00 执行）
   .\scripts\setup-task.ps1
   ```

### 定时规则

- **执行时间**: 每 6 小时执行一次 (0:00, 6:00, 12:00, 18:00)
- **执行命令**: `docker compose up -d`
- **阅读时长**: 68 分钟 (由 WEREAD_DURATION 控制)

### 工作原理

1. Windows 任务计划程序定时触发
2. 执行 `docker compose up -d` 启动容器
3. 容器启动后会自动阅读 68 分钟
4. 阅读完成后容器保持运行状态

### 验证定时任务

```powershell
# 方式1: PowerShell
Get-ScheduledTask | Where-Object {$_.TaskName -like "*WeRead*"}

# 方式2: schtasks
schtasks /Query /FO TABLE | findstr "WeRead"
```

### 删除定时任务

```powershell
# 删除所有 WeRead 定时任务
schtasks /Delete /TN "WeRead-3016" /F
schtasks /Delete /TN "WeRead-3016-06" /F
schtasks /Delete /TN "WeRead-3016-12" /F
schtasks /Delete /TN "WeRead-3016-18" /F
```

### 手动执行

```powershell
# 立即执行
cd E:\dockers\weread-challenge-selenium-3016
docker compose up -d
```

## 调试记录

- Windows 任务计划程序不支持单任务多触发器（每6小时）
- 解决方案：创建4个独立任务（0:00, 6:00, 12:00, 18:00）
- 任务名格式：WeRead-3016, WeRead-3016-06, WeRead-3016-12, WeRead-3016-18

## 注意事项

- 需要以管理员身份运行 PowerShell
- 首次部署需要扫码登录（二维码保存在 data/login.png）
- 确保 Docker Desktop 正常运行
