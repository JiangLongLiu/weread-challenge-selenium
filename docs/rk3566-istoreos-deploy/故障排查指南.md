# 故障排查指南

## 常见问题与解决方案

### 1. SSH 连接失败

#### 问题描述
无法通过 SSH 连接到 RK3566 主机。

#### 可能原因
- 网络不通
- SSH 服务未启动
- 密码错误

#### 解决方案
1. 检查网络连通性：
```bash
ping 192.168.123.51
```

2. 检查 SSH 服务状态（需要在主机上执行）：
```bash
# 登录路由器或通过其他方式
# 检查 SSH 配置
```

3. 确认密码正确，检查 password.csv 文件

### 2. 容器启动失败

#### 问题描述
docker compose up -d 执行后容器未启动。

#### 可能原因
- Docker 资源不足
- 镜像拉取失败
- 端口冲突

#### 解决方案
1. 查看日志：
```bash
docker compose -f /mnt/sata1-1/docker/mycontainers/weread-challenge-selenium/docker-compose.yml logs
```

2. 检查磁盘空间：
```bash
df -h
```

3. 检查端口占用：
```bash
netstat -tuln | grep -E '4444|7900'
```

### 3. Selenium 健康检查失败

#### 问题描述
Selenium 容器健康检查未通过。

#### 可能原因
- 内存不足
- Chrome 启动失败

#### 解决方案
1. 增加共享内存大小，确保 shm_size 至少 2GB
2. 检查容器日志：
```bash
docker compose logs selenium
```

3. 重启 Selenium 容器：
```bash
docker compose restart selenium
```

### 4. 登录二维码未生成

#### 问题描述
data/login.png 文件未生成。

#### 可能原因
- 容器未正常启动
- 网络问题无法访问微信读书

#### 解决方案
1. 检查应用日志：
```bash
docker compose logs app | grep -i "qr"
```

2. 检查网络连接：
```bash
# 在容器内测试网络
docker exec -it weread-challenge-selenium-app-1 ping weread.qq.com
```

### 5. 阅读时间不计数

#### 问题描述
微信读书显示的阅读时间与实际不符。

#### 解决方案
- 确保每日阅读时间达到 68 分钟以上（推荐 65+ 分钟）
- 检查是否开启截图功能
- 确认网络稳定，在线阅读

### 6. Cookies 过期

#### 问题描述
需要重新扫描二维码登录。

#### 解决方案
1. 删除旧的 cookies：
```bash
rm /mnt/sata1-1/docker/mycontainers/weread-challenge-selenium/data/cookies.json
```

2. 重启容器生成新二维码：
```bash
docker compose restart app
```

3. 扫描新二维码登录

### 7. VNC 界面无法访问

#### 问题描述
无法访问 http://192.168.123.51:7900

#### 可能原因
- 端口未映射
- 防火墙阻止
- 容器未运行

#### 解决方案
1. 确认端口映射存在：
```bash
docker ps | grep selenium
```

2. 检查防火墙规则：
```bash
# 在 iStoreOS 上检查
iptables -L -n
```

3. 重启 Selenium 容器

## 诊断命令汇总

```bash
# 1. 查看所有容器状态
docker ps -a

# 2. 查看容器日志
docker compose logs -f

# 3. 查看资源使用
docker stats

# 4. 检查网络
curl http://192.168.123.51:4444/status

# 5. 进入容器调试
docker exec -it weread-challenge-selenium-app-1 /bin/sh
```
