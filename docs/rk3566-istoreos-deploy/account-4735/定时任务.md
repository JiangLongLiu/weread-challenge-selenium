# 定时任务配置 (account-4735)

本文件说明如何配置 account-4735 的定时阅读任务。

## 方案：crontab 定时任务

### 配置步骤

1. 上传配置到 RK3566：
   - `docker-compose.yml`
   - `scripts/setup-cron.sh`

2. 在 RK3566 上执行：
   ```bash
   # 添加执行权限
   chmod +x scripts/setup-cron.sh

   # 运行定时任务配置脚本
   bash scripts/setup-cron.sh
   ```

### 定时规则

- **执行时间**: 每 6 小时执行一次 (0:00, 6:00, 12:00, 18:00)
- **执行命令**: `docker compose up -d`
- **阅读时长**: 68 分钟 (由 WEREAD_DURATION 控制)

### 工作原理

1. crontab 每天 00:00 执行 `docker compose up -d`
2. docker-compose.yml 中配置了 `restart: unless-stopped`
3. 容器启动后会自动阅读 68 分钟
4. 阅读完成后容器保持运行状态（等待下次重启）

### 验证定时任务

```bash
# 查看当前定时任务
crontab -l

# 查看容器状态
docker ps | grep account-4735
```

### 删除定时任务

```bash
crontab -e
# 删除 account-4735 相关的行
```

## 注意事项

- 确保 Docker 服务正常运行
- 首次部署需要扫码登录（二维码保存在 data/login.png）
- selenium 容器也配置了 restart，会一直运行
